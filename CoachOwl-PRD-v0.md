# CoachOwl — 产品需求文档 (PRD) v0.2

> 面向澳洲市场的轻量级私教/教练管理 SaaS
> 定位：低成本、易上手的管理工具（非获客 marketplace）
> 状态：草稿 v0.2 · 待 Aurora 评审
> 关键决策（已定）：邮件提醒优先，SMS 后置为付费钩子；通知层做成 channel-agnostic；首发 AI 楔子＝智能导入。

---

## 1. 背景与问题

独立 tutor / coach（语言、学科辅导、音乐、健身、技能培训）和小型工作室，目前普遍用 Excel + 短信/微信/邮件手动管理学员、排课和收款。痛点集中在：

- 排课、调课、请假靠人工记录，容易冲突或漏排
- 课时包/余额靠手算，对账麻烦，纠纷多
- 收款分散（现金、转账、第三方），月底统计费时
- 没有统一的学员档案和上课记录
- 提醒靠手动发消息，No-show 率高

现有成熟产品（TutorBird、TutorCruncher、My Music Staff 等）功能全但偏贵、偏重，对**单人或 2–5 人小团队**而言是杀鸡用牛刀，学习成本和月费都偏高。

## 2. 目标与非目标

**目标（MVP）**
- 让一个独立私教在 10 分钟内完成注册→建学员→排第一节课
- 把"排课 + 课时/余额 + 收款记录 + 自动提醒"这条主线做扎实
- 月费做到明显低于主流竞品，主打"够用、不贵、不复杂"

**非目标（MVP 暂不做）**
- 双端 marketplace / 获客撮合（明确排除）
- 复杂的多校区 / 多角色权限体系
- 内置视频会议（用 Zoom/Meet 链接字段即可）
- 全功能财务/工资/税务系统（只做基础收支记录与导出）

## 3. 目标用户

| 角色 | 描述 | 核心诉求 |
|------|------|----------|
| 独立私教（主用户） | 单人运营，10–40 名学员 | 省时、不漏课、按时收钱 |
| 小工作室主理人 | 2–5 名 tutor/coach | 统一排课、看整体收入 |
| 学员 / 家长（次要） | 接收提醒、查看课表、可选自助预约 | 知道什么时候上课、还剩几节 |

> 澳洲本地化要点：货币 AUD、GST（10%）可选开关、日期格式 DD/MM/YYYY、时区按用户设置（默认 Australia/Sydney）。

## 4. MVP 功能范围

按 MoSCoW 分级。

### Must have（MVP 核心闭环）

1. **账户与上手**
   - 邮箱注册/登录，单一 owner 账户
   - 引导式 onboarding：建第一个学员 → 排第一节课

2. **学员管理（CRM-lite）**
   - 学员档案：姓名、联系方式、家长信息（未成年）、备注、标签
   - 状态：活跃 / 暂停 / 流失
   - **智能导入**：CSV / 粘贴表格 → AI 自动识别字段、解析重复课程文本（详见 §8，首发 AI 楔子）

3. **排课与日历**
   - 单次 & 重复课程（每周固定）
   - 周/月视图，冲突检测
   - 调课、请假、取消（记录原因）

4. **课时包 / 余额**
   - 学员购买 N 节课，上一节扣一节
   - 余额低提醒（如剩 ≤2 节）

5. **收款记录与发票**
   - 手动记录付款（现金/转账/第三方），对应课时包或单次
   - 生成简单 PDF 发票（含可选 GST）
   - 收入概览：本月已收 / 待收

6. **自动提醒**
   - 课前提醒（学员/家长），**MVP 仅邮件**；SMS 后置为付费 add-on（成本原因见 §7）
   - 余额不足提醒
   - 通知层做成 channel-agnostic：统一发送接口，底层先接邮件，后续挂 SMS / 推送 / WhatsApp adapter 不动业务逻辑

### Should have（紧随 MVP）

- 学员/家长只读课表页（链接访问，无需登录）
- 上课记录 / 进度笔记（每节课一条）
- 基础报表：出勤率、月收入趋势、No-show 统计
- 多 tutor 支持（工作室场景，简单角色：owner / coach）

### Could have（后续）

- 学员自助预约（开放空档时段）
- 在线收款集成（Stripe，AUD）
- 课时包模板 / 批量定价
- 与 Google Calendar 双向同步

### Won't have（明确不做）

- Marketplace 撮合 / SEO 获客
- 复杂权限、审批流
- 内置视频通话

## 5. 关键用户流程

**流程 A：私教首次上手**
注册 → 设置时区/货币/GST → 新建学员 → 创建重复课程（每周二 16:00）→ 设置课时包（10 节 $50/节）→ 系统自动按周生成课程并扣课时。

**流程 B：处理一次调课**
日历点开某节课 → 改时间 / 标记请假 → 选择是否扣课时 → 自动通知学员/家长。

**流程 C：月底对账**
收入概览 → 查看本月已收/待收 → 对未付学员一键发提醒/发票。

## 6. 非功能需求

- **简单优先**：核心操作 ≤3 次点击；移动端友好（私教常在路上）
- **性能**：常规列表/日历加载 <1s（单租户数据量小）
- **数据隔离**：多租户，行级隔离
- **可靠性**：提醒类任务幂等、可重试，不重复发送
- **邮件送达率**：邮件是 MVP 唯一触达通道，必须配好 SPF / DKIM / DMARC，用事务型服务发送，避免进垃圾箱
- **通知可扩展**：channel-agnostic 抽象，新增通道＝加 adapter，不改业务逻辑
- **合规**：澳洲隐私（Privacy Act / APPs），未成年人家长信息单独标注；数据可导出/删除
- **可维护**：单人维护得动（你一个人 side project，架构别太重）

## 7. 成本与商业模式（"低成本"是核心约束）

**成本结构（关键洞察）**
- 免费版的边际成本几乎全在通道上。**只要免费层不含 SMS，每个活跃用户的成本接近零。**
- **计算 + 托管 Postgres**：$0–25 AUD/月（hobby 档可撑数百用户，是总额非人均）
- **事务邮件**：≈ $0.10 USD / 1000 封（SES）；一个 20 学员的活跃免费用户每月约 $0.02 AUD
- **SMS（唯一失血点）**：澳洲本地 3–8 分 AUD/条。贵的部分是运营商落地费（三大运营商寡头定价），不是聚合商加价——自建网关绕不开，故 SMS 一律放付费层

> 监管提示：澳洲 SMS Sender ID Register 自 2026-07-01 生效。要让提醒显示已验证的 "CoachOwl" 品牌名（而非被标 "Unverified"），需用 AURORA INNOLABS 的 ABN 注册 sender ID。注册已开放、处理需数周——一旦启用 SMS 通道，提前办。CoachOwl 的定位是**注册品牌 + 转售聚合商短信**，不做运营商/CSP。

**付费钩子（埋点）**
按转化力，建议主计费轴 + 两个独立功能开关：

| 钩子 | 类型 | 逻辑 |
|------|------|------|
| **活跃学员数上限**（主轴） | 价值锚 | 免费 ≤5 学员；带 30 学员的私教月入可观，升级几乎无痛 |
| **SMS 提醒** | 成本对齐 | 免费发邮件，SMS 付费；同时挡住失血点 |
| **在线收款 / Stripe** | 赚钱才付费 | 免费手动记账，付费才能让学员 app 内付款 |
| 多 tutor 席位 | 工作室 | 免费单人，付费多教练 |
| 自助预约 / 报表导出 / 去品牌水印 | 便利/品牌 | 锦上添花层 |

**定价方向（待定）**
- Free（永久）：1 tutor、≤5 学员、邮件提醒、手动记账
- Solo：约 $9–15 AUD/月，无限学员、邮件提醒
- Studio：约 $25–40 AUD/月，多 tutor、报表、SMS 额度、在线收款
- 对标：明显低于 TutorCruncher / TutorBird 同档位

## 8. AI 功能

定位原则：**最好的 AI 嵌在核心流程里，而非单独贴标签。** 按真实 ROI 排序，标注复杂度与是否进 MVP。

| # | 功能 | 价值 | 复杂度 | 阶段 |
|---|------|------|--------|------|
| 1 | **智能导入**（首发楔子） | 决定"进得来" | 中 | **MVP** |
| 2 | **课后笔记 / 语音笔记助手**（留存核心） | 决定"用得住" | 中 | **MVP（轻量版）** |
| 3 | 自然语言排课 | 绕开繁琐排课 UI | 中 | Phase 2 |
| 4 | 自动起草沟通 | 省情绪劳动 | 低 | Phase 2 |
| 5 | 槽位建议（"智能排课"轻量形态） | 找补课空档 | 低–中 | Phase 2 |
| 6 | No-show / 流失预测 | 主动挽留 | 中 | Phase 3（需历史数据） |
| 7 | 收款对账模糊匹配 | 减轻月底对账 | 低 | Phase 3 |

**1. 智能导入（MVP · 首发差异化楔子）**
私教的表格通常很乱：列顺序随机、姓名和电话混在一格、上课时间是"周二周四 4-5pm"这类自由文本。LLM 直接吃任意 CSV / 粘贴内容 → 自动识别字段、拆分姓名/联系方式/家长、把"每周二四 4pm"解析成结构化重复课程。命中"采纳那一刻"的摩擦，是 AI 的甜区（乱的非结构化 → 结构化）。

**2. 课后笔记 / 语音笔记助手（MVP 轻量版 · 日常留存）**
课后教练随手打字或语音输入几句 → AI 整理成结构化进度笔记，跨课追踪主题，期末一键生成给家长的进度总结。语音→笔记尤其适合在路上的教练（本地 faster-whisper 转写，Aurora 已有此栈）。MVP 先做"文本/语音 → 结构化笔记"，期末总结可放紧随其后。

**3. 自然语言排课（Phase 2）**
"把我周二的孩子下周都挪到周三"、"Emma 这学期每周一 4pm 固定" → 解析成日历操作。排课 UI 天生繁琐，NL 输入绕开大部分点击。

**4. 自动起草沟通（Phase 2）**
根据学员档案生成续费提醒、期末进度邮件、调课通知等，语气得体、个性化。复杂度低，可与 §2 笔记复用同一套上下文。

**5. 槽位建议（Phase 2 · 取代"全局智能排课"）**
"给 X 补一节，大家都合适的空档是哪个？" —— 轻量的槽位推荐，而非全局最优排课。
> 说明：真正的"自动最优排课"是约束求解问题（OR-tools 比 LLM 更对），但独立私教多为固定周课表、极少全量重排，受众与频率都窄，MVP ROI 差。故降级为槽位建议，求解器明确后置或不做。

**6. No-show / 流失预测（Phase 3）**
出勤下滑、余额长期不用、出现空档 → 标记高流失风险，提示教练主动联系。依赖历史数据，待数据积累后做。

**7. 收款对账模糊匹配（Phase 3 · niche）**
银行转账备注模糊匹配到学员（"JOHN SMITH TUTOR" → 学员档案），减轻月底对账。

**实现备注**
- LLM 调用集中在结构化抽取与文本生成，走外部 API 即可；语音转写可用本地 faster-whisper（Aurora 的 RTX 3080 Ti）控成本
- AI 功能成本（尤其转写/长上下文）也可作为付费层的隐性边界
- 所有 AI 输出在写库前给用户**确认/可编辑**一步，避免错误静默落库（导入、笔记尤其重要）

## 9. 技术考量（轻量,供参考)

- 后端契合你现有栈：Python / FastAPI + Postgres
- 提醒走异步任务队列（如 ARQ / Celery + Redis 或轻量 cron）
- 邮件用事务型服务（如 Resend / SES），配齐 SPF/DKIM/DMARC；SMS 后置，用聚合商（ClickSend/MessageMedia/Twilio）+ 注册 sender ID
- AI：结构化抽取/生成走外部 LLM API；语音转写用本地 faster-whisper（RTX 3080 Ti）
- 通知层 channel-agnostic：统一 send 接口 + 各通道 adapter
- 前端可先做响应式 Web，移动端体验优先；原生 App 后置
- 多租户：单库 + tenant_id 行级隔离起步，够用

> 技术栈仅为方向建议,具体实现等 PRD 定稿后再细化,不在本文展开代码。

## 10. 成功指标（MVP 验证）

- 激活率：注册后完成"建学员 + 排第一节课"的比例 >50%
- 留存：第 4 周仍在排课的账户比例
- 核心价值证明：用户用它管理的学员数 / 课程数月增长
- 付费转化：试用→付费比例（验证"低价但有人付"）
- AI 楔子有效性：智能导入完成率（开始导入→成功建档）、笔记功能周活跃使用率

## 11. 开放问题（需 Aurora 决策）

1. **首发垂直**：先聚焦哪一类私教？（语言/学科 vs 音乐 vs 健身——功能细节差异在课时、群课、器材等）
2. **群课**：MVP 是否支持一对多/小班，还是只做 1:1？（影响排课和扣课时模型）
3. **学员是否需要登录**：MVP 用只读链接够吗，还是要做学员账户？
4. **在线收款**：MVP 先只做手动记录，Stripe 放后续？
5. **命名最终确认**：CoachOwl 需正式核查澳洲商标 + 域名（coachowl.com / .com.au）可用性——上次只是初步搜索。
